# NeuMathe React Viewer

An interactive, high-performance web application for browsing and studying math questions scraped from NeuMathe.

NeuMathe 本地刷题站前端 - 一个用于浏览和练习数学题目的交互式 Web 应用。

---

## ✨ Features / 功能特性

- **🌳 Interactive Chapter Tree**: Navigate through complex question categories with a recursive tree structure.
  - **交互式目录树**: 通过递归树结构轻松导航复杂的题目分类。
- **🧪 LaTeX & Markdown Rendering**: Superior math typesetting using KaTeX and Markdown.
  - **LaTeX & Markdown 渲染**: 使用 KaTeX 和 Markdown 实现卓越的数学公式排版。
- **⭐ Favorites & Progress**: Save difficult questions and track your study progress using LocalStorage.
  - **收藏与进度**: 使用 LocalStorage 收藏难题并自动记录学习进度。
- **🔍 Intelligent Filtering**: Filter questions by difficulty scores, accuracy rates, and time spent.
  - **智能筛选**: 按难度分、正确率及平均耗时筛选题目。
- **🌓 Dark/Light Mode**: Seamlessly switch between themes for comfortable night-time studying.
  - **暗夜模式**: 支持一键切换主题，保护视力。
- **📱 Responsive Layout**: Optimized for both desktop and mobile viewing with a "Hero" header and clean panel layout.
  - **响应式布局**: 针对桌面端和移动端优化的布局设计。

---

## 🛠️ Technology Stack / 技术栈

- **Core**: React 19, Vite, TypeScript
- **Routing**: React Router 7
- **Math Rendering**: KaTeX, remark-math, rehype-katex
- **Markdown**: React Markdown
- **Data Fetching**: Native Fetch API (consuming local JSON bundles)
- **State Management**: React Hooks (useState, useMemo, useEffect)

## 📂 Data Structure / 数据组织

The application expects its data to be located in `public/data/`:
应用的数据应放置于 `public/data/` 目录下：

- `neumathe_chapter_tree.json`: The full metadata for the chapter navigation.
- `neumathe_chapter_{id}_raw.json`: The actual question content for each specific chapter.

*Note: These files are typically generated by the companion Python scraper in the parent directory.*
*注：这些文件通常由父目录中的配套 Python 爬虫生成。*

### 📊 Chapter Coverage / 章节覆盖率
The current version is optimized for chapters **256 through 400** (Probability and Statistics).
当前版本针对第 **256 至 400 章节**（概率统计）进行了优化。

## 🖥️ VPS Deployment Guide (Ubuntu) / VPS 部署教程（Ubuntu）

This guide walks you through deploying the production build to an Ubuntu VPS using **Nginx** as the web server.

本教程将引导你使用 **Nginx** 将生产构建部署到 Ubuntu VPS。

---

### Step 1: Build the Project Locally / 第一步：本地构建

在本地项目目录下执行构建命令，生成静态产物：

```bash
# 在 neumathe-react/ 目录下执行
npm install
npm run build
```

构建完成后，所有静态文件会输出到 `dist/` 目录。

> **注意**：`public/data/` 下的数据文件会自动被复制到 `dist/data/`，无需额外处理。

---

### Step 2: Connect to Your VPS / 第二步：连接到 VPS

```bash
ssh your_user@your_vps_ip
# 例如：ssh root@123.45.67.89
```

---

### Step 3: Install Nginx on the VPS / 第三步：在 VPS 上安装 Nginx

```bash
sudo apt update
sudo apt install -y nginx
# 启动 Nginx 并设置开机自启
sudo systemctl start nginx
sudo systemctl enable nginx
```

验证 Nginx 是否正常运行：

```bash
sudo systemctl status nginx
```

---

### Step 4: Upload Build Files to VPS / 第四步：上传构建产物到 VPS

**方法一：使用 `rsync`（推荐，增量同步快）**

在本地机器上执行：

```bash
# 将 dist/ 内容同步到 VPS
rsync -avz --delete ./dist/ your_user@your_vps_ip:/var/www/neumathe/
```

**方法二：使用 `scp`（一次性上传）**

```bash
scp -r ./dist/* your_user@your_vps_ip:/var/www/neumathe/
```

**在 VPS 上创建目标目录（如尚不存在）：**

```bash
sudo mkdir -p /var/www/neumathe
sudo chown -R $USER:$USER /var/www/neumathe
```

---

### Step 5: Configure Nginx / 第五步：配置 Nginx

在 VPS 上创建 Nginx 站点配置文件：

```bash
sudo nano /etc/nginx/sites-available/neumathe
```

粘贴以下配置内容（将 `your_domain_or_ip` 替换为你的域名或 IP）：

```nginx
server {
    listen 80;
    server_name your_domain_or_ip;

    root /var/www/neumathe;
    index index.html;

    # 支持 React Router 的 SPA 刷新（fallback 到 index.html）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源长期缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 数据文件不缓存（方便更新）
    location /data/ {
        expires -1;
        add_header Cache-Control "no-cache, no-store, must-revalidate";
    }

    # 开启 Gzip 压缩，加快传输速度
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml image/svg+xml;
    gzip_min_length 1024;
}
```

启用该配置并重启 Nginx：

```bash
# 创建软链接启用站点
sudo ln -s /etc/nginx/sites-available/neumathe /etc/nginx/sites-enabled/

# 测试配置语法
sudo nginx -t

# 重新加载 Nginx
sudo systemctl reload nginx
```

现在你应该可以通过 `http://your_domain_or_ip` 访问站点了。

---

### Step 6 (Optional): Enable HTTPS with Let's Encrypt / 第六步（可选）：配置 HTTPS

> **前提**：你必须拥有一个已解析到该 VPS IP 的域名。

```bash
# 安装 Certbot 和 Nginx 插件
sudo apt install -y certbot python3-certbot-nginx

# 自动申请并配置 SSL 证书（替换为你的域名和邮箱）
sudo certbot --nginx -d your.domain.com --email you@example.com --agree-tos --non-interactive

# 测试证书自动续期
sudo certbot renew --dry-run
```

Certbot 会自动修改 Nginx 配置，将 HTTP 重定向到 HTTPS，并设置证书自动续期（通过 systemd timer）。

---

### Step 7: Update Data Files / 第七步：更新数据文件

每次抓取新数据后，只需重新上传 `data/` 目录，无需重新构建：

```bash
# 本地数据在 public/data/ 里，直接同步到 VPS
rsync -avz --delete ../public/data/ your_user@your_vps_ip:/var/www/neumathe/data/
```
